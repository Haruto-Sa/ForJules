# コーディネート提案画面 ワイヤーフレーム

## 1. ページ概要

**目的**: AIがユーザーのワードローブを基に、最適なコーディネートを提案する画面

**主要機能**:
- 条件指定（場面、天気、スタイル）
- AI提案結果表示
- 提案の保存・評価
- 代替アイテム提案
- SNSトレンド連携

## 3. 機能詳細

### 3.1 条件設定
- **場面選択**: 仕事、デート、カジュアル、フォーマル、スポーツ
- **天気設定**: 晴れ、曇り、雨、雪、気温範囲
- **スタイル指定**: カジュアル、フォーマル、エレガント、スポーティ
- **季節考慮**: 自動検出 or 手動選択

### 3.2 AI提案システム
- **メイン提案**: 最も適したコーディネート1つ
- **信頼度表示**: アルゴリズムの確信度を%で表示
- **代替案**: 2-3つの追加オプション
- **理由説明**: なぜそのコーディネートを提案したかの簡単な説明

### 3.3 アイテム置換機能
- **個別変更**: 各アイテムを別のもので置換
- **類似アイテム**: 同じカテゴリの代替案表示
- **リアルタイム更新**: 変更後の全体バランス再計算

### 3.4 評価・保存機能
- **評価**: いいね/イマイチでフィードバック
- **保存**: 気に入ったコーディネートを保存
- **履歴**: 過去の提案と評価の確認

## 4. インタラクションフロー

### 初回利用時
```
条件設定 → AI分析中 → 結果表示 → 評価・保存
    ↓
   ワードローブ不足の場合
    ↓
 [アイテム追加画面へ]
```

### リピート利用時
```
前回条件引継ぎ → 微調整 → 新しい提案 → 評価・保存
```

### アイテム不足時
```
提案生成失敗 → 不足アイテム表示 → 購入提案 → ワードローブ追加
```

## 5. レスポンシブ対応

### モバイル（<768px）
- 条件選択を縦並び
- アイテム詳細をカルーセル表示
- スワイプで代替提案切り替え

### タブレット（768px-1024px）
- 2列レイアウト（条件 | 結果）
- アイテム詳細をグリッド表示

### デスクトップ（>1024px）
- 3列レイアウト（条件 | メイン結果 | 代替案）
- 全体を一画面で表示

## 7. 技術実装メモ

### コンポーネント構成
- `OutfitGenerator`: メイン提案コンポーネント
- `ConditionSelector`: 条件選択UI
- `OutfitDisplay`: 提案結果表示
- `ItemSwapper`: アイテム置換機能
- `SuggestionCard`: 代替案カード

### API連携
- `POST /api/outfit_suggestions`: 提案生成
- `POST /api/outfits`: 提案保存
- `PUT /api/outfits/{id}/rating`: 評価送信

### 状態管理
- 提案条件の管理
- 生成中の状態管理
- ユーザー評価の追跡
- 保存済み提案の管理

### パフォーマンス最適化
- 提案結果のキャッシュ
- 画像のプリロード
- バックグラウンドでの代替案生成
